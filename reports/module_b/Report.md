### Описание работы системы:
В системе реализована 2 контура обработки пакетный (для глубокой работы с историческими данными)
и потоковой (для работы в реальном времени)

### 1) Файл: services\streaming\producer.py - 
Точка входа для потокового контура, именно тут выполняется захват видеопотока и формирование сообщений в consumer
А также запись каждого кадра, который удовлетворяет условию: frames % FRAMES_PER_SEC, в MinIO хранилище функция: send_frame_to_s3

```
	VIDEO_PATH - Переменная, хранит в себе относительный путь к видеопотоку
	CAMERA_ID - ID камеры, на которую идет фиксация
	DIRECTION - Направление, тип данных Float, сейчас стоит заглушка
	WIDTH - Ширина, (заглушка)
	LENGTH - Долгота, (заглушка)
	FRAMES_PER_SEC - Кол-во кадров в секунду на видеопотоке
	TOPIC - тема, в которую Kafka будет фиксировать сообщения
	BUCKET_NAME - Имя бакета, куда фиксируются кадры детекции
	PX_PER_METER - Сколько метров в 1 пикселе (Сейчас взято заглушкой, необходимои поменять значение когда известна колибровка камеры)
```

### 2) Файл: services\streaming\consumer.py - Чтение сообщений из TOPIC, json сериализация, запись в raw_layer.raw_detections
Каждые POLL_TIMEOUT секунд Consumer читает сообщения из TOPIC.
Вставка в таблицу raw_layer.raw_detections происходит по нескольким условиям:

1) Сообщения не вставляеются сразу, как consumer их считал, 
	собирается список из INSERT_CONST кол-во сообщений, после чего вставляется в таблицу и очищается
2) Если в списке собранных сообщений есть сообщения, но их не INSERT_CONST, а время с последней вставки прошло > TIME_TO_INSERT_SEC, то тогда выполняется вставка в таблицу

```
	POLL_TIMEOUT = 5
	INSERT_CONST = 300
	TIME_TO_INSERT_SEC = 5
```

### 3) Соотвествующие SQL / python функции для агрегации данных и формировании витрин.
1) services\batch\transform_raw_to_staging.py - Функция, которая высчитывает ключевые показатели, такие как:
- Выборка детекций где уверенность модели в распозновании > 0.5
- Направление (south, north, west, east)
- Скорость трека в km/h services\batch\utils\calculate_speed.py (Единая функция для расчета скорости) используется в обоих контурах обработки.
2) services\batch\build_hourly_traffic.py - функция, которая агегирует данные из staging_layer.detections_clean, 
	используется в обоих контурах. Формирует витрину hourly_traffic
3) services\batch\load_csv_to_raw.py - функция, загружающая данные из CSV в таблицу raw_layer.csv_traffic_raw
4) services\batch\transform_csv_to_staging.py - функция, которая переводит данные из raw_layer.csv_traffic_raw в staging_layer.detections_clean (Единую версию правды)
5) services\batch\build_realtime_traffic.py - функция, предназначенная для записи данных в витрину mart_layer.realtime_traffic, агрегирует записи по 5 минут за окно.
6) database\DWH - медальонная архитектура DWH хранилища
